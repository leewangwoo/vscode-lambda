# LiteLLM v2 + LLM Monitor 배포 가이드

## 개요

- **LiteLLM 1.93.0**: reasoning_content 전달 지원 (기존 1.10.1은 strip함)
- **LLM Monitor**: 모든 LLM 인스턴스의 실시간 busy/available 상태 제공 (사이드카)
- **VS Code 확장 999.41.0**: reasoning 표시 + 상태 표시줄 + maxTokens 설정

## 서버로 전송할 파일

```
litellm_config/
├── airgap/
│   └── litellm-airgap-v2.tar          ← LiteLLM 1.93.0 이미지 (344MB)
├── llm-monitor/
│   └── llm-monitor.tar                ← LLM Monitor 이미지 (50MB)
└── SLTPRODL01/
    ├── docker-compose-v2.yaml         ← 배포용 docker-compose
    ├── litellm-config-v2.yaml         ← LiteLLM 설정
    ├── monitor-config.py              ← Monitor 설정 (서버용)
    └── 배포가이드-v2.md               ← 이 파일
```

## 배포 순서

### 1. 이미지 로드

```bash
# 두 tar 파일을 서버의 SLTPRODL01 폴더로 전송 후
docker load -i ../airgap/litellm-airgap-v2.tar
docker load -i ../llm-monitor/llm-monitor.tar

# 확인
docker images | grep -E "litellm-airgap-v2|llm-monitor"
```

### 2. 테스트 실행 (포트 8089/8090)

기존 LiteLLM(8088)은 그대로 두고 새 버전을 테스트합니다:

```bash
cd /path/to/SLTPRODL01

# 새 버전 실행 (기존 LLM 인스턴스는 자동으로 재사용)
docker compose -f docker-compose-v2.yaml up -d
```

### 3. 동작 확인

```bash
# LiteLLM 모델 목록 + reasoning_content 확인
curl -s http://localhost:8089/v1/models -H "Authorization: Bearer dummy-key"
# → {"data":[{"id":"qwen27",...},{"id":"qwen35",...}]}

# reasoning_content 전달 확인
curl -s http://localhost:8089/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer dummy-key" \
  -d '{"model":"qwen27","messages":[{"role":"user","content":"hi"}],"stream":true,"max_tokens":100}' | head -5
# → reasoning_content 필드가 보여야 함

# LLM Monitor 상태 확인
curl -s http://localhost:8090/health
# → {"total":10,"available":10,"busy":0,"offline":0}

curl -s http://localhost:8090/status | python3 -m json.tool
# → 각 인스턴스별 상세 상태
```

### 4. VS Code 설정

내부망 PC의 VS Code 설정:

```json
{
    "github.copilot.chat.byok.customoai": {
        "url": "http://100.252.201.200:8089/v1",
        "key": "dummy-key",
        "maxTokens": 4096
    }
}
```

확인 항목:
- 채팅 모델 선택기에 Qwen 모델 표시
- 질문 시 Reasoning 섹션에 내용 표시
- 하단 상태 표시줄에 `✅ LLM: 10 사용 가능` 표시
- 채팅 중 `⏳ LLM: 9 가능 / 1 처리중` 으로 변경

### 5. 정식 전환 (포트 8088로)

테스트 완료 후 기존 LiteLLM을 교체:

```bash
# 기존 LiteLLM 중지
docker stop litellm

# docker-compose-v2.yaml 수정:
#   litellm-v2의 포트를 8089 → 8088로 변경
#   container_name을 litellm-v2-test → litellm로 변경
# 그리고 재시작
docker compose -f docker-compose-v2.yaml up -d
```

## 문제 해결

### 모델이 안 보이는 경우
```bash
# LiteLLM 로그 확인
docker logs litellm-v2-test --tail 20
```

### reasoning이 안 나오는 경우
- LiteLLM 버전 확인: `docker exec litellm-v2-test pip show litellm | grep Version`
- 1.93.0이 아니면 이미지를 다시 로드

### Monitor가 offline만 표시하는 경우
- Docker 네트워크 확인: `docker network inspect litellm-stack_llm-net`
- LLM 인스턴스들이 같은 네트워크에 있는지 확인
- monitor-config.py의 URL이 Docker 내부 호스트네임과 일치하는지 확인
